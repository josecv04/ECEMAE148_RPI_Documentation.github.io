Install Raspberry Pi Imager

Download and launch the official flashing tool for your Raspberry Pi

Raspberry Pi Imager is the official tool for writing OS images to a microSD card. It handles downloading the OS, verifying the image, writing it to your card, and applying customizations like hostname, WiFi, and SSH ‚Äî all in one place. Always use the most current version.

üö®
Critical ‚Äî Use Legacy 64-bit Only
You must install Raspberry Pi OS (Legacy, 64-bit) ‚Äî the Debian Bookworm port. This ships with Python 3.11 and is the only version compatible with DonkeyCar 5.2.x.

Do NOT install Debian Trixie. Trixie ships with Python 3.13, which breaks DonkeyCar and is the root cause of all camera issues reported in this class.
1
Download Raspberry Pi Imager
Navigate to the official Raspberry Pi software page and download the latest Imager for your operating system (Windows, macOS, or Linux). Install it normally.
Official download URL
https://www.raspberrypi.com/software/
2
Insert Your microSD Card
Insert your 128 GB microSD card into your computer using a USB hub, card adapter, or built-in slot ‚Äî whichever works best for your machine. The card must be detected before launching the Imager.
3
Select Device
Open Raspberry Pi Imager. Under Device, choose Raspberry Pi 5 (or whichever model you have). This ensures the correct kernel and firmware are used.
SettingValueWhy
DeviceRaspberry Pi 5Selects the correct ARM64 kernel and firmware
OSRaspberry Pi OS (Legacy, 64-bit)Python 3.11 ‚Äî required for DonkeyCar 5.2.x
StorageYour 128 GB microSD cardTarget device for the OS image
02

Flash the OS ‚Äî Choose the Correct Image

Select Raspberry Pi OS Legacy 64-bit (Bookworm) ‚Äî not Trixie
Critical

After selecting your device, click OS to choose the operating system. You must scroll down to find the Legacy version. The default recommended image at the top is Trixie (Python 3.13) and will break DonkeyCar.

‚ö†
Python Version Matters
Raspberry Pi OS (Legacy, 64-bit) ‚Üí Python 3.11 ‚Üí DonkeyCar 5.2.x works ‚úì
Raspberry Pi OS (Debian Trixie) ‚Üí Python 3.13 ‚Üí DonkeyCar broken ‚úó / camera broken ‚úó

Why the Legacy Image?

DonkeyCar 5.2.x depends on several Python packages whose C extensions are only compiled against Python 3.11. Attempting to install them on Python 3.13 will either fail entirely or produce silent runtime errors that manifest as camera failures ‚Äî the most common issue reported in this class.

05

Enable SSH & Write the Image

Enable remote access, then safely flash and eject the microSD card

Enable SSH

On the Remote Access tab, toggle Enable SSH on and select Use password authentication. Without this, you cannot connect to your Pi over the network without a keyboard and monitor.

Raspberry Pi Connect (Optional)

The Raspberry Pi Connect tab offers a cloud-based remote access service. This is optional for this class ‚Äî skip it unless you specifically need remote desktop access outside the local network.

Review and Write

After completing all customisation tabs, you will see a summary screen showing all your choices. Verify everything looks correct, then click WRITE.

Write Complete ‚Äî Eject Safely

When writing finishes, the Imager will show a success screen. Click Finish, then follow this exact ejection order to avoid corrupting the card:

1
Eject via OS First
Use your operating system's eject function (right-click ‚Üí Eject on macOS/Windows, or umount on Linux) to safely flush all pending writes. Do not physically remove the card yet.
2
Physically Remove the Card
Only after the OS confirms it is safe, carefully remove the microSD from the adapter, hub, or computer slot. Skipping the software eject risks corrupting the OS files you just flashed.
üö®
Do Not Skip the Software Eject
Pulling the card without a software eject is the most common cause of corrupted OS installations. If the card is corrupt you will need to re-flash from scratch. Always eject through your OS first.
06

Cooling & Hardware Assembly

Install the cooler, case, and microSD before first boot

The Raspberry Pi 5 runs hotter than previous models under sustained workloads. The active cooler (fan + heatsink) is required, not optional. Install it before powering on for the first time.

Step-by-Step Assembly

1
Clean the Board Surface
If previous teams have handled the board, clean the chip surfaces with isopropyl alcohol wipes before applying new thermal pads. Residue from previous thermal compound will reduce thermal conductivity.
2
Remove Thermal Pad Plastic Film
Each thermal pad ships with a thin plastic protective film on both sides. Remove both films before mounting the cooler. Leaving the plastic on is a common mistake ‚Äî the pad will not conduct heat through the plastic.
3
Connect the Fan Cable
Remove the fan connector cover on the Pi board. Connect the fan cable ‚Äî there is only one correct orientation, do not force it. The connector is keyed so it only inserts one way.
4
Install into Case
Place the Pi into the case, aligning all port cutouts. Apply the rubber feet. Secure with the provided hardware. Install the cover. Verify all connectors (USB, HDMI, Ethernet, power) are accessible through the case openings.
5
Insert the microSD Card
With the Pi in its case, insert the freshly flashed microSD card into the SD card slot. Ensure it clicks firmly into place.
‚Ñπ
Power Supply Requirements
For initial configuration, use the provided USB-C 5A power supply. Any 40W+ USB-C PD supply will work. The Pi may show a GUI warning about power not being 5V 5A ‚Äî this is expected and harmless. The Pi is looking for a PD signal not all supplies provide. This setting can be adjusted later in the Pi configuration.
07

First Boot & SSH Connection

Power on the Pi and connect remotely over WiFi

On first boot, the Pi performs initial OS configuration ‚Äî expanding the filesystem, setting up user accounts, and applying all the customisations you set in the Imager. This can take several minutes. The green LED will flash as it accesses the SD card, and the Pi may reboot once during this process. Be patient.

Connect via SSH

Once the Pi has finished its initial boot and connected to your WiFi network, you can SSH in from any device on the same network. Replace XX with your team number.

Your laptop terminal
ssh pi@ucsdrobocar-148-XX.local

# Example for Team 5:
ssh pi@ucsdrobocar-148-05.local
‚Ñπ
How .local Hostnames Work
The .local suffix uses mDNS (Bonjour/Avahi) to resolve the hostname to an IP address automatically. This works as long as both your laptop and the Pi are on the same WiFi network. You do not need to know the Pi's IP address ‚Äî just its hostname.

First Connection Security Prompt

The first time you SSH to a new Pi, your terminal will ask you to verify the host fingerprint. Type yes to accept and add it to your known hosts file. This is normal and expected ‚Äî it won't ask again for this Pi.

Expected first-connection prompt
# The authenticity of host 'ucsdrobocar-148-05.local' can't be established.
# ECDSA key fingerprint is SHA256:xxxxxxxxxxxxxxxxxxxx.
# Are you sure you want to continue connecting (yes/no/[fingerprint])? yes

08

WiFi Configuration

List available networks, add additional access points, and find your IP

Once connected via SSH, you can manage WiFi networks from the command line. The Pi uses NetworkManager, so you can use either the text UI (nmtui) or the command-line tool (nmcli).

List Available WiFi Networks

Pi terminal
sudo nmcli device wifi list   # List all visible networks

Add a New WiFi Network via nmtui (Recommended)

The nmtui text UI is the easiest way to add new networks. Launch it and choose Activate a connection to connect to a visible network, or Edit a connection to add one manually (e.g., a hidden SSID).

Pi terminal
sudo nmtui

Add a Network via Command Line (Alternative)

Pi terminal
# Connect to a WiFi network by SSID and password
sudo nmcli device wifi connect "NETWORK_NAME" password "NETWORK_PASSWORD"

# Class network reference:
# SSID: UCSDRoboCar
# Password: UCSDrobocars2018
sudo nmcli device wifi connect "UCSDRoboCar" password "UCSDrobocars2018"

Get the Pi's IP Address

After connecting, you can find the Pi's assigned IP address. Use ifconfig and look at the wlan0 interface ‚Äî the inet field shows the IPv4 address.

Pi terminal
ifconfig
# Look for wlan0 ‚Üí inet field (e.g., 192.168.x.x)

Install nano Text Editor

While not strictly required, nano is a simple terminal text editor useful for editing config files. Install it now as it will be needed in later steps.

Pi terminal
sudo apt-get update
sudo apt-get install nano
09

System Settings

Change hostname, password, and system configuration with raspi-config

If you need to change the hostname or password after flashing ‚Äî or if you inherited a Pi from a previous team ‚Äî use raspi-config or the commands below. Always follow the class hostname standard.

Change Hostname with raspi-config

Launch the Raspberry Pi Software Configuration Tool. Navigate to S4 Hostname (under System Options) and set your hostname to the class standard.

Pi terminal
sudo raspi-config
# Navigate: System Options ‚Üí S4 Hostname
# Set to: ucsdrobocar-148-XX (replace XX with your team number)

Change Hostname via Command Line

Alternatively, you can set the hostname directly from the terminal. You must update it in two places ‚Äî hostnamectl and /etc/hosts ‚Äî otherwise the Pi will show "unable to resolve host" errors on every sudo command.

Pi terminal ‚Äî replace xxx-yy with your class and team number
# Step 1: Set the active hostname
sudo hostnamectl set-hostname ucsdrobocar-xxx-yy

# Step 2: Update the hosts file (change the 127.0.1.1 line)
sudo nano /etc/hosts
# Change the line that reads: 127.0.1.1  old-hostname
# To:                          127.0.1.1  ucsdrobocar-xxx-yy
# Save with Ctrl-X ‚Üí Y ‚Üí Enter
‚ö†
Update Both Files
If you only run hostnamectl set-hostname without updating /etc/hosts, every sudo command will print a warning: "unable to resolve host ucsdrobocar-xxx-yy". This does not break anything, but it is distracting and slows down terminal output.

Change Password

Pi terminal
passwd   # Changes the password for the current user (pi)

View Current System Information

Pi terminal
hostnamectl   # Shows hostname, OS, kernel, architecture

Reboot to Apply All Changes

Pi terminal
sudo reboot now

Date & Time Configuration

Ensure the clock is correct or software installs will fail

If the Pi's clock is wrong ‚Äî which happens when it has been offline or when the battery-backed RTC is not present ‚Äî apt will reject package installations because SSL certificates will appear expired. Always check the date before debugging any apt errors.

‚ö†
Check Date First When Updates Fail
If apt update or pip install fails with SSL or certificate errors, the clock is almost certainly wrong. Check and correct the date before asking for help or debugging further.

Check the Current Date

Pi terminal
date   # Prints current system date and time

Manually Set Date and Time

If the Pi is not connected to the internet or NTP is not running, set the time manually using the format below. Replace the timestamp with the current date and time.

Pi terminal
sudo date -s '2025-09-15 10:30:00'   # Replace with current date/time

Set the Timezone

Pi terminal
sudo dpkg-reconfigure tzdata
# Navigate: America ‚Üí Los_Angeles

Install and Force NTP Time Sync

NTP (Network Time Protocol) automatically synchronises the clock from internet time servers. Install it once and it will keep the clock correct whenever the Pi has internet access.

Pi terminal
sudo apt-get update
sudo apt-get install ntp

# Force an immediate time sync:
sudo /etc/init.d/ntp stop
sudo ntpd -q -g
sudo /etc/init.d/ntp start

# Reboot to confirm time persists correctly:
sudo reboot now
‚úì
Force Sync Anytime
If the time drifts later (e.g., after the Pi was offline for a long time), you can force a re-sync without reinstalling: sudo ntpd -q -g

GUI Settings & Filesystem Expansion

Disable the desktop GUI to save RAM, and expand the filesystem to use the full SD card

Disable the GUI on Boot (Optional but Recommended)

Since you interact with the Pi exclusively over SSH, the desktop GUI wastes CPU cycles and RAM. Disabling it frees up significant resources for DonkeyCar's neural network inference. You can always re-enable it if you need graphical debugging (e.g., for RealSense cameras or LiDAR).

‚Ñπ
Note on GUI for Sensor Debugging
When using Intel RealSense cameras, LiDAR, or other sensors that require ROS visualisation tools (e.g., RViz), you may need to keep the GUI (X Server) running. For headless DonkeyCar operation, disabling it is safe.
Pi terminal
sudo raspi-config
# Navigate: System Options ‚Üí S5 Boot ‚Üí B1 Console Text console
# This sets the Pi to boot to a text console instead of the desktop

Expand the Filesystem

The OS image is smaller than your 128 GB card. By default, the Pi only uses the space the image occupied (~2‚Äì4 GB). You must expand the filesystem to use all available storage. Without this, you will run out of space when installing DonkeyCar and its models.

Pi terminal ‚Äî option 1: raspi-config
sudo raspi-config
# Navigate: Advanced Options ‚Üí A1 Expand Filesystem

Update and Upgrade the System

After expanding the filesystem and configuring the Pi, run a full system update. This installs the latest security patches and ensures all package lists are current before you install DonkeyCar.

Pi terminal
sudo apt update      # Refresh package lists
sudo apt upgrade     # Install available updates (say Y when prompted)
sudo reboot now      # Reboot to apply kernel updates if any
‚ö†
Do NOT Run apt full-upgrade
sudo apt full-upgrade can upgrade the kernel and firmware to versions that break hardware compatibility. Use sudo apt upgrade (without "full") to install only safe package-level updates.

Passwordless SSH (Key-Based Auth)

Set up SSH key pairs to connect without typing a password every time
Optional

Key-based SSH authentication is more secure and more convenient than passwords ‚Äî you never type a password, and your private key cannot be brute-forced over the network. This is strongly recommended if you SSH into your Pi frequently during development.

Step 1 ‚Äî Generate a Key Pair on Your Laptop

Your laptop terminal
# Install rsync if not already installed (Linux/WSL)
sudo apt-get install rsync

# Generate an RSA key pair ‚Äî press Enter to accept default location
ssh-keygen -t rsa

Step 2 ‚Äî View Your Public Key

Your laptop terminal
# macOS / Linux:
cat ~/.ssh/id_rsa.pub

# Ubuntu / WSL:
cat .ssh/id_rsa.pub

# Copy the entire output ‚Äî it starts with "ssh-rsa AAAA..."

Step 3 ‚Äî Add the Key to the Pi's Authorized Keys

SSH into the Pi and create the .ssh directory if it doesn't exist, then paste your public key into the authorized_keys file.

Pi terminal (SSH into Pi first)
cd ~
mkdir .ssh          # Safe to ignore "File exists" error
chmod 700 .ssh
sudo chown pi:pi .ssh
nano .ssh/authorized_keys
# Paste your public key here (the entire ssh-rsa AAAA... line)
# Save: Ctrl-X ‚Üí Y ‚Üí Enter
chmod 600 .ssh/authorized_keys

Step 4 ‚Äî Test the Connection

Your laptop terminal
ssh pi@ucsdrobocar-148-XX.local
# Should connect WITHOUT asking for a password
‚úì
Faster Workflow
Once key-based auth is working, tools like scp and rsync also work without passwords ‚Äî making it easy to transfer trained DonkeyCar models and data logs between your laptop and the Pi.

Backup & Troubleshooting

Back up your configured SD card and fix the most common setup issues
üíæ
Back Up Your SD Card Now
You have spent several hours configuring this Pi. Before proceeding to DonkeyCar installation, create a full backup image of your SD card. If anything goes wrong during software installation, you can restore to this working state in minutes instead of starting over.

How to Back Up (Raspberry Pi Imager)

Shut down the Pi, remove the SD card, and insert it into your laptop. Open Raspberry Pi Imager. Under OS, scroll down to Use custom ‚Äî but instead of writing to the card, use your OS's disk imaging tool (dd on macOS/Linux, Win32DiskImager on Windows) to create a .img file from the card.
